RSA oracle Attack

发表于 2018-07-02 更新于 2022-03-17 分类于 crypto

这篇笔记介绍在密码学中著名的Bleicherbacher Orcale攻击。Bleicherbacher Orcale攻击针对RSA加密及签名方案中，使用固定填充方式带来的实现中的安全问题。为解决这类问题，后续推荐使用带随机性的填充方式如PSS和OAEP来代替传统的PKCS填充。

1998年密码学家Daniel Bleichenbacher发现在SSL协议中[1]，使用PKCS#1 v1.5填充方式的实现，会面临Adaptive-chosen ciphertext攻击。攻击者可以通过将一系列精心选择的密文发送给SSL协议server，通过分析server的不同相应，在不获取私钥的情况下，可以破解目标密文对应的明文。

在2006年，Bleichenbacher发布了对于PKCS#1 v1.5填充方式的RSA签名算法，当padding解析存在缺陷的情况下，对于很小的公钥e=3，可以通过
简单的计算获取对任意数据的签名，该漏洞存在在openssl实现中。

本文包含攻击原理解释，以及对应的python代码实现。内容保含如下典型RSA Oracle攻击方法

  • Parity Oracle Attack
  • Bleichenbacher attack on Encryption(98)
  • ROBOT (Return Of Bleichenbacher oracle aTtack)
