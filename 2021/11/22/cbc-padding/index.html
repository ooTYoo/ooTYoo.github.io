CBC Padding Oracle 攻击

发表于 2021-11-22

0x01. Padding Oracle

在密码学中的padding oracle attack 是一种通过padding是否有限这一信息来破解密文的攻击方式。在加解密计算中，使用的加解密算法通常对输入长度有特定要求，当输入不满足要求时，需要使用特定的填充方法将输入补齐到需要的长度。

以AES-128分组密码算法为例，AES-128bit 处理的输入块是128bit，通常的待加密明文按照分组长度划分后，剩余部分内部不满足长度要求，此时需要padding将剩余部分填充到128bit。常见的填充方式有两种：

  1. CMS padding：又称PKCS#7 padding，设剩余输入长度为L比特，则需要补充n个byte，n=BLOCK_SIZE - L，每个填充的byte值为n。例如剩余输入为 b'hello' ，AES-128对应的BLOCK_SIZE=16 Byte，则填充11个0xb, padding后的尾部为 b'hello\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b'
  2. ISO 9797 padding：补充‘0x80’ + (n-1) 个‘0x00’ 。还以剩余输入为 b'hello' 为例，该方式padding后的尾部为b'hello\x80\0x00\0x00\0x00\0x00\0x00\0x00\0x00\0x00\0x00\0x00

在攻击者控制密文的情况下，虽然无法直接获得明文，但有时可以获得 明文中padding是否正确这一信息。如从服务器端回复的报错信息 参考ncc文章给出的例子。 如同一个黑盒机器，攻击者将密文放入，机器回答“Yes/No”。这样的机器我们成为Oracle。利用Oracle给出的信息展开的密码学攻击成为Oracle Attack。

初看Oracle给出的信息很有限，貌似攻击者能作的事情不多，但在一些情况下，仅使用这些信息就足够攻击者恢复全部的明文内容。本文介绍的分组密码算法中CBC模式的Padding Oracle, 就可以获取全部的明文内容。其他的类似攻击方式后续将诸如分析完善。

0x03. CBC Padding Oracle

针对CBC模式的Padding Oracle Attack属于chosen cipher-text attack范畴。以CMS Padding为例分析CBC解密过程，攻击者可以将k+1 个分组之后的密文做截断，通过第k个分组中引入的单Byte错误，改变第k+1组明文的单个Byte，该分组被服务器当作最后一组，即padding分组。获取明文Pt_k最后1Byte的攻击步骤如下：

  1. 修改第k个分组的最后1个Byte为0x00，

  2. 此时Ct_k变为Ct_k’，将 Ct_0||Ct_1||…|| Ct_k’ || Ct_{k+1} 送入Oracle

  3. 如果Oracle回答“No”, 修改最后1个Byte为0x01，执行第2步，直到Oracle回答“Yes”

  4. Oracle回答“Yes”,表明最后1Byte作为padding是合法的。则明文Pt_{k+1}必然是如下形式之一

    • xxxxxx 0x01
    • xxxxxx 0x02 0x02
    • xxxxxx 0x03 0x03 0x03
    • …
    • 0x10 0x10 0x10 … 0x10

  5. 在LSB被修改值的基础上，修改倒数第2个Byte为任意其他值，如果Oracle返回No，则回到第3步继续执行

  6. 如果Oracle 返回 Yes，可以确定此时的Pt_{k+1}’ 最后1Byte被修改为0x1

  7. 利用如下关系获得Pt_{k+1}最后1Byte的值

    Ptk+1′=Dec(key,Ctk+1)⊕Ctk′=...0x01Dec(key,Ctk+1)=Ctk′⊕...0x01Ptk+1=Dec(key,Ctk+1)⊕Ctk=Ctk′⊕Ctk⊕...0x01Pt_{k+1}'= Dec(key, Ct_{k+1}) \oplus Ct_{k}' = {...0x01}\\ Dec(key, Ct_{k+1}) = Ct_{k}'\oplus {...0x01}\\ Pt_{k+1}= Dec(key, Ct_{k+1}) \oplus Ct_{k} = Ct_{k}'\oplus Ct_{k}\oplus {...0x01} Ptk+1′​=Dec(key,Ctk+1​)⊕Ctk′​=...0x01Dec(key,Ctk+1​)=Ctk′​⊕...0x01Ptk+1​=Dec(key,Ctk+1​)⊕Ctk​=Ctk′​⊕Ctk​⊕...0x01

获得最后1Byte之后，可以使用类似的思路攻击倒数第2Byte，并以此类推，获取第k+1分组的明文。最终恢复出所有分组对应的全部明文。

0x05. 如何防御

直觉的防御办法是消除oracle的存在，但这却是没有看起来那么容易，oracle不仅可以通过服务器的错误返回信息来构造，还可以通过server返回的时间侧信道信息构造，想要完全消除这种信息泄漏是件较为困难的事情。

oracle攻击可以实施的一个重要条件是server会对攻击者输入的任何信息作计算，这个问题可以通过对输入的信息在处理前现进行校验来解决。如使用带教研机制的AES-GCM模式。
