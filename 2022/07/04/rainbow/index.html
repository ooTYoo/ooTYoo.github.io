0x1 多变量公钥密码算法（MPKC）

Rainbow算法属于多变量公钥密码算法（MPKC, Multivariate Public Key Cryptography）的一种。MPKC基于的困难问题是“在有限域上求解多变量非线性多项式组”。MPKC中所用的多项式为二次多项式（有研究表明更高次数多项式反而会降低系统安全性）。MPKC中通常定义m个多项式 p(1)…p(m)p^{(1)}\ldots p^{(m)}p(1)…p(m), 其中每个多项式为n个变量 $x_1, x_2,\ldots, x_n $ 的二次多项式。

其中所有的系数和变量取值都在有限域 Fq\mathbb{F}_qFq​ 上。 当为n个变量 $ x_i$ 输入特定的值之后，这m个多项式求值可以得到 m个结果。

P=(y1,y2,…,ym)=(p(1)(x1,…,xn),…,p(m)(x1,…,xn))\mathcal{P} = (y_1,y_2,\ldots,y_m) = (p^{(1)}(x_1,\ldots,x_n),\ldots,p^{(m)}(x_1,\ldots,x_n)) P=(y1​,y2​,…,ym​)=(p(1)(x1​,…,xn​),…,p(m)(x1​,…,xn​))

MPKC的困难问题定义如下：

MQ Problem : 给定m个多项式的结构，求解n个解 xˉi\bar{x}_{i}xˉi​ , 使得

p(1)(xˉ1,…,xˉn)=p(2)(xˉ1,…,xˉn)=…=p(m)(xˉ1,…,xˉn)=0p^{(1)}(\bar{x}_{1},\ldots,\bar{x}_{n})=p^{(2)}(\bar{x}_{1},\ldots,\bar{x}_{n})=\ldots =p^{(m)}(\bar{x}_{1},\ldots,\bar{x}_{n}) = 0 p(1)(xˉ1​,…,xˉn​)=p(2)(xˉ1​,…,xˉn​)=…=p(m)(xˉ1​,…,xˉn​)=0

一般性的MQ问题被证明在GF上是NP困难的。且当前不存在能够有效求解MQ问题的量子算法。

MPKC需要在一般性的MQ的基础上添加trapdoor来构造公私钥对。其核心想法是，利用一个可以很容易求解的 多项式映射（central map） F\mathcal{F}F , 通过在其前后复合可逆的Affine map S,TS, TS,T, 得到如下关键要素

  • 公钥 $ \mathcal{P}$ , 其为复合映射

P=S⋅F⋅T\mathcal{P} = S\cdot \mathcal{F} \cdot T P=S⋅F⋅T

  • 私钥为 F,S,T\mathcal{F}, S, TF,S,T
  • 解密/签名过程：给定m个结果(y1,…yn)(y_1,\ldots y_n)(y1​,…yn​), 对 P\mathcal{P}P 求逆得到一组解(x1,…,xn)(x_1,\ldots,x_n)(x1​,…,xn​)
  • 加密/验签过程：给定n个输入$(x_1, x_2,\ldots, x_n) $, 对多项式组求值

整体过程如下图所示

由于非线性映射不是一一映射，对m和n的选取有如下限制：加解密方案中要求 m≥nm\geq nm≥n,签名方案中要求 m≤nm\leq nm≤n 。对应于解密过程明文唯一，签名场景下签名存在。签名场景下可能存在其他的解，同一份消息可能有多个签名都能通过验签。

另外，trapdoor形式的MQ问题求难是否是NP问题当前尚无证明。

0x2. OV方案

Rainbow算法是基于所谓的OV（Oil and Vinegar）方案。该方案通过巧妙的将n个编码分为两组来构造trapdoor，即所谓的O和V组。该方案中多项式不存在O组中变量的二次耦合，该特性是构造可逆central map的关键。

  • V组 (x1,…,xv)(x_1,\ldots, x_v)(x1​,…,xv​)， 共 v 个变量

  • O\组 (xv+1,…,xn)(x_{v+1},\ldots, x_n)(xv+1​,…,xn​), 共o个变量，$ o = n - v$

  • 所用的多项式形似如下

    f(x)=∑i,j∈Vαi,jxixj+∑i∈V,j∈Oβi,jxixj+∑iγixi+ηf(x) = \sum_{i,j\in V} \alpha_{i,j}x_i x_j + \sum_{i\in V,j\in O} \beta_{i,j}x_i x_j + \sum_i \gamma_i x_i + \eta f(x)=i,j∈V∑​αi,j​xi​xj​+i∈V,j∈O∑​βi,j​xi​xj​+i∑​γi​xi​+η

Cnetral Map 求逆的算法思想如下

    1. 给定m个结果 (y1,…,ym)(y_1,\ldots, y_m)(y1​,…,ym​)

    2. 选取v个随机的x1,…,xvx_1,\ldots, x_vx1​,…,xv​ 带入m个多项式中，将o个剩余的变量x作为变量，得到一组线性方程组。

      f(1)=∑j∈Oβj′(1)xj+η′(1)=y1…f(m)=∑j∈Oβj′(m)xj+η′(m)=ymf^{(1)} = \sum_{j\in O} \beta'^{(1)}_j x_j + \eta'^{(1)} = y_1\\ \ldots\\ f^{(m)} = \sum_{j\in O} \beta'^{(m)}_j x_j + \eta'^{(m)} = y_m f(1)=j∈O∑​βj′(1)​xj​+η′(1)=y1​…f(m)=j∈O∑​βj′(m)​xj​+η′(m)=ym​

    3. 使用高斯消元发求解方程组，得到o个(xv+1,…,xn)(x_{v+1},\ldots, x_n)(xv+1​,…,xn​)

    4. 如果3中求解失败，则返回2中重新产生前v个随机输入。直至3中求解成功

    5. 求逆的结果为 (x1,…,xn)(x_{1},\ldots, x_n)(x1​,…,xn​)

原始OV方案中O和V两组变量数目相当，该方便被invariant subspace attack 攻破，之后方案改进为UOV （unbalanced OV）。Rainbow方案相当于一个多层的UOV方案。

Rainbow方案中为u层UOV方案。选取如下分层参数

1<v1<v2<...<vk<…<vu<vu+1=n1 < v_1 <v_2 <...<v_k < \ldots < v_u < v_{u+1} = n 1<v1​<v2​<...<vk​<…<vu​<vu+1​=n

定义的u+1中分组方式如下

\begin{align} V_i &= \{x_1,\ldots, x_{v_{i}}\},\quad |V_i|=v_i\\ O_i &= V_{i+1} - V_i = \{x_{v_{i}+1},\ldots, x_{v_{i+1}}\}, |O_i| = o_i = v_{i+1} - v_{i} \end{align}

Central Map F\mathcal{F}F 由 m=n−v1m = n - v_1m=n−v1​ 个多项式组成，编号分别为

\begin{align} F^{O_1} &= \{f^{(v_{1}+1)},\ldots, f^{(v_{2})}\} \\ \ldots\\ F^{O_k}& = \{f^{(v_{k}+1)},\ldots, f^{(v_{k+1})}\} = \{f^{(k)} | k \in O_k\}\\ \ldots\\ F^{O_u}& = \{f^{(v_{u}+1)},\ldots, f^{(n)}\} = \{f^{(k)} | k \in O_u\}\\ \ldots\\ \mathcal{F} &= F^{O_1}\cup \ldots\cup F^{O_u} = \{f^{v_1+1},\ldots,f^{n}\} \end{align}

由于每个O分组中包含不止一个变量。PQC提交材料中将指标k所在的分组记为OlO_lOl​, 对应的多项式定义为

f(k)(x)=∑i,j∈Vlαi,j(k)xixj+∑i∈Vl,j∈Olβi,j(k)xixj+∑i∈Vl∪Olγi(k)xi+η(k)f^{(k)}(x) = \sum_{i,j\in V_l} \alpha^{(k)}_{i,j}x_i x_j + \sum_{i\in V_l,j\in O_l} \beta^{(k)}_{i,j}x_i x_j + \sum_{i\in V_l\cup O_l} \gamma^{(k)}_i x_i + \eta^{(k)} f(k)(x)=i,j∈Vl​∑​αi,j(k)​xi​xj​+i∈Vl​,j∈Ol​∑​βi,j(k)​xi​xj​+i∈Vl​∪Ol​∑​γi(k)​xi​+η(k)

0x3 Rainbow 签名方案详解

0x 3.1 参数

  • 有限域 Fq\mathbb{F}_qFq​
  • 映射参数n，m,
  • 分层参数 v1=n−m,v2,…,vuv_1 = n-m, v_2,\ldots, v_uv1​=n−m,v2​,…,vu​

当前提交的算法中使用的分层参数为u=2, 方案用于签名和验签。u>2 对安全性增强不大。

0x 3.2 key Generation

私钥为 S，T, F\mathcal{F}F 中的参数

公钥为 $\mathcal{P} $ 中的参数

需要注意的是，其中防射变换T是对n个输入的变量进行，而防射变换S则是对m个多项式的求值结果进行。两者发生在不同的 空间中。

密钥派生计算对应

  • 生成有限域上#sk个随机数
  • 有限域上的矩阵运算（矩阵乘法）

0x3.3 签名方案

签名方案中大的算子为：

  • GF上矩阵乘法、求逆、加减运算
  • GF上高斯消元法求根（也可以看作是一种矩阵求逆方法）

0x3.4 验签

签名方案中大的算子为：

  • 多项式求值（GF上矩阵乘法、加运算）pdf中写为MQ evaluation
